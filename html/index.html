Web Dev Concepts

Web Servers

As it relates to websites and servers, the index is the first page that a browser will request from a server (assuming a request for a specific resource isn't made). For example, when visiting this site, you'll be served the index.html file that resides on this webserver. From this page, you can access the rest of the content hosted here.

Frontend Design

Frontend design describes the graphical interface of a website and experience of interacting with it. A good frontend design facilitates easy navigation of a web page. It should also include a consistent color scheme, font and typography. Any visual media such as images or video should also fit into the design scheme. While there are ideas that have held the test of time, trends in frontend design are always changing so it's best to keep on your toes.

The 5 E's of Usability

Effectiveness:
helps users meet their goal(s) and provide accurate results
Effeciency:
perform tasks with as few steps as possible; provide results quickly
Ease of navigation
users (especially new ones) should quickly understand how to navigate toward their goal by interacting with the site; the navigation process is memorable
Error-free
avoid accessibility/availability issues; learn from common UX errors users experience
Enjoyable or Engaging
the site is enjoyable to use or engaging; the content and design fits target audience needs such that they return to use it again

Layout Tags

  • <header>: sets the banner for a web page
    • info within usually includes the name of the page, the publisher, and a marketing slogan
    • often remains consistent across pages to idenfify them as part of the main site
    • Smaller headers are preferred so that there's more room for the main content
  • <nav> tag is used to denote the navigation section of the site, where users can go to other pages of content
  • <main>: the primary block that contains the main content of the page
  • <section>: used for the thematic grouping of content; usually followed by an <h1> element to describe it
  • <article>: typically nested in a <section> element; used to contain a single topic with an <h2> element as its title
  • <aside>: used for content related to an article; may also be used for sidebars or unrelated content
  • <figure>: used for media; typically paired with a <figcaption> element
  • <blockquote>: used for extended quotations, stylized by default to stand out
  • <footer>: holds legal and contact information; links to critical pages; usually contains the copyright statement
  • <div>: placeholder element for dynamic content; has no style or dimensions
  1. With an anchor (<a>) element, you can link to external content using URLs such as https://google.com for the tag's href attribute.
  2. Similarly, you can link to content within a page using CSS IDs
  3. You can link to content on other pages using URIs or relative pathnames to resources on a webserver. This page's navbar is an example of the latter.
